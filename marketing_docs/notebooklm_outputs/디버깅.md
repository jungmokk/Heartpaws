Heartpaws 앱의 프론트엔드 디버깅을 위해 집중해야 할 체크리스트를 5가지 핵심 영역으로 나누어 정리해 드립니다. 특히 최근 진행한 프로필 UI 개편과 **펫 꾸미기 기능(ProfileDecorator)**을 중심으로 실무적인 디버깅 포인트를 짚었습니다.

1. 사용자 상호작용 및 제스처 디버깅 (Interactive UI)
가장 복잡한 로직이 포함된 

ProfileDecorator
 컴포넌트의 동작을 검증해야 합니다.

다중 액세서리 제어: 여러 아이템이 겹쳐있을 때 특정 아이템이 올바르게 선택(selectedIndex)되는지, 선택되지 않은 아이템의 제스처가 간섭을 일으키지 않는지 확인하세요.
좌표 정규화 (Normalization): 스티커의 위치값($x, y$)이 절대 좌표가 아닌 0~1 사이의 비율로 저장되고 렌더링 시 CONTAINER_SIZE에 맞춰 정확히 복원되는지 디버깅이 필요합니다.
리사이즈 및 회전 한계값: 액세서리가 너무 커지거나 작아져서 사라지는 현상이 없는지, 최소/최대 스케일(0.2 ~ 3.0) 및 회전 각도가 부드럽게 적용되는지 확인하세요.
2. 상태 동기화 및 렌더링 (State & Rendering)
NotebookLM에서도 언급된 useFocusEffect와 관련된 데이터 흐름을 체크해야 합니다.

상태 업데이트 레이스 컨디션: 

handleEquipFromProfile
 호출 시 DB 업데이트와 로컬 상태(dbProfile, pendingAccessories) 업데이트 사이의 시차가 사용자에게 "반영 안 됨"으로 느껴지지 않는지 로그를 찍어 확인하세요.
Memoization 효율성: useMemo로 계산되는 equippedItems와 loadoutItems가 불필요하게 자주 재계산되지는 않는지 혹은 종속성 배열(deps)이 누락되어 업데이트가 안 되는지 검토가 필요합니다.
데코레이션 모드 전환: editingAccessory 상태가 바뀔 때 전체 레이아웃이 급격하게 흔들리거나(Layout Shift), ScrollView의 스크롤 잠금(scrollEnabled)이 정확히 작동하는지 확인하세요.
3. 백엔드 연동 및 예외 처리 (API & Error Handling)
Supabase RPC 및 Storage 연동 과정에서의 에러를 잡아내야 합니다.

RPC 응답 처리: save_accessory_data, feed_virtual_pet 등의 RPC 호출 실패 시 사용자에게 알림(Alert)이 정확히 가는지, 그리고 로컬 UI가 실패 이전 상태로 롤백되는지 확인하세요.
이미지 업로드 프로세스: 아바타 사진 변경 시 ImageManipulator가 압축한 결과물이 Supabase Storage에 정상 업로드되는지, 네트워크 지연 시 ActivityIndicator가 사용자에게 충분한 피드백을 주는지 디버깅하세요.
Null Safety: dbProfile이나 user 객체가 순간적으로 null이 될 때 발생하는 TypeError(예: Cannot read property 'avatar_url' of null)를 방지하기 위한 Optional Chaining(?.)이 모든 곳에 적용되었는지 전수 조사해야 합니다.
4. 레이아웃 및 디자인 일관성 (Layout & Aesthetics)
StitchMCP 시안을 적용한 후의 시각적 완성도를 디버깅합니다.

Safe Area Insets: 기기마다 다른 상단 노치(Notch)와 하단 바 영역이 헤더의 paddingTop과 겹치지 않는지 확인하세요.
Glassmorphism 효과: 투명 배경(rgba)과 테두리(borderWidth)가 적용된 피딩 바와 로드아웃 바가 다크 모드 배경에서 시인성이 충분한지 디버깅이 필요합니다.
반응형 대응: 화면 너비가 좁은 기기에서 좌/우 정보(경험치 바와 사용자 정보)가 중앙 아바타를 침범하지 않는지 확인하세요.
5. 성능 및 빌드 이슈 (Performance & Build)
Metro 캐시 이슈: 코드 수정 후 npx expo run:android에서 반영이 안 되는 현상은 대부분 캐시 문제입니다. npx expo start -c로 캐시를 날리고 재시도하는 과정을 자동화하거나 체크리스트에 넣으세요.
Reanimated 성능: 액세서리를 드래그할 때 프레임 드랍(Stuttering)이 없는지 모니터링하세요. (특히 runOnJS(true)로 인한 메인 스레드 부하 여부 확인)
